# Maestro Roadmap

語言：[English](./ROADMAP.md) · [简体中文](./ROADMAP.zh-CN.md) · [繁體中文](./ROADMAP.zh-TW.md) · [更多語言](./LANGUAGES.md)

## 目標

Maestro 的目標很簡單：

> **讓 AI Agent 更容易、更安全、更可靠地進入真實工程團隊。**

很多 Coding Agent 已經會寫程式碼。但團隊真正需要的不只是程式碼生成：

- 工作應該來自 TAPD、Linear 以及未來更多真實專案系統；
- 程式碼應該來自明確設定的 Git 倉庫和分支；
- 每次執行都應該有獨立工作區，避免不同任務互相干擾；
- 人應該能看懂 Agent 做了什麼、改了什麼、為什麼失敗；
- 高風險步驟應該保持可審查；
- 團隊應該可以逐步擴大使用，而不是第一天就打開所有權限。

這份路線圖按使用者價值組織，而不是按內部模組名稱組織。

---

## 近期：讓 Maestro 更容易試用

第一次接觸 Maestro 的使用者，應該不需要先理解完整架構，也能看懂並跑起來。

計畫包括：

- 更簡單的本地 demo；
- 更清楚的 Quick Start；
- 截圖、GIF 或短影片；
- 能展示完整流程的範例任務；
- 更清楚地說明獨立工作區的價值：並行、隔離、清理和可審查；
- 說清楚仍然存在的 `symphony` 相容命名；
- 從本地 demo 走到真實專案設定的清晰路徑。

我們希望更容易展示的場景包括：

- TAPD 任務到 GitHub Pull Request；
- Linear 任務到 GitHub Pull Request；
- 寫程式前的需求分析；
- 新任務分診；
- reviewer 建議；
- 在相似任務上比較 Codex、Claude Code 和 OpenCode。

成功標準是：新讀者可以在幾分鐘內回答：

>「Maestro 做什麼？我的團隊為什麼可能需要它？」

---

## 下一階段：讓 Agent 接入真實專案流程

Maestro 應該讓 Agent 從團隊已經在使用的專案系統開始工作，而不是要求團隊另外建立新的任務佇列。

計畫包括：

- 改善目前 TAPD 和 Linear 流程；
- 讓任務狀態、評論、連結和結果更容易理解；
- 讓 workflow template 更容易尋找、複製和修改；
- 支援更多常見研發任務：Bug 修復、小功能、需求分析、任務澄清、任務分診和評審建議；
- 明確區分目前已支援的接入能力和未來可擴展目標；
- 為 Jira、GitHub Issues、GitLab、Gitea、Bitbucket、飛書專案等更多系統做好準備。

成功標準是：團隊可以從自己現有的專案流程出發，而不必為了使用 Agent 改變管理工作的方式。

---

## 中期：讓 Agent 執行更可信

團隊不應該只因為 Agent 說「完成了」就相信一次執行。

計畫包括：

- 更清楚的執行歷史；
- 更容易閱讀的執行摘要；
- 任務、Git 變更、日誌和評審資料之間更好的連結；
- 更清楚的失敗原因；
- 更好的日誌脫敏；
- 更有用的 dashboard；
- 在寫回真實專案系統、推送分支或建立 PR 前提供可見檢查點；
- 更清楚地區分本地 demo、可信評估、團隊試點和生產運行。

成功標準是：reviewer 可以回答：

- Agent 做了什麼？
- 它基於哪個任務和 Git 倉庫工作？
- 它改了什麼？
- 它為什麼停下來？
- 還有哪些需要人工確認？
- 現在繼續是否安全？

---

## 長期：幫助團隊規模化使用 Agent

單個 Agent demo 有價值，但團隊級使用需要更強的運營能力。

計畫包括：

- 安全地同時執行多個任務；
- 為不同專案和任務保留獨立工作區與執行紀錄；
- 依照任務類型選擇不同 Agent；
- 更清楚地管理帳號、憑據、額度和成本；
- 改善團隊級執行環境；
- 提供更好的重試和恢復；
- 支援更清楚的人工審批點；
- 幫助團隊比較不同 Agent 和流程的實際效果。

成功標準是：團隊可以在控制安全、成本和品質的前提下，逐步擴大 Agent 使用範圍。

## 文件與社群

Maestro 應該先讓人看懂，再讓人感受到強大。

計畫包括：

- 保持主 README 簡短且以例子為主；
- 把更深入的技術細節放到獨立文件；
- 積極維護英文和簡體中文；
- 保留其他語言翻譯，並歡迎社群改進；
- 增加專案系統、Agent、程式碼平台和 workflow template 的貢獻指南；
- 發布更多真實工程場景範例。

成功標準是：貢獻者不需要先讀完整個程式碼庫，也能找到有用的切入點。

---

## 目前暫不追求

Maestro 不試圖幫團隊繞過 review、測試或發布判斷。

我們更重視：

- 把 Agent 連接到真實任務；
- 讓程式碼來源和任務來源可見；
- 讓執行過程可追蹤；
- 在高風險步驟保留人的控制；
- 保留有用的執行紀錄；
- 只在信任逐步累積後擴大自動化。

自動化應該隨著證據擴大，而不是隨著想像擴大。

---

## 目前重點

目前重點是讓 Maestro 更容易理解、更容易試用，也更安全地進行評估：

1. 簡化公開 README；
2. 增加通俗的 Roadmap；
3. 改善本地 demo 指引；
4. 說明目前接入能力，但不把外部系統說成「內建」；
5. 說清楚獨立工作區的價值；
6. 增加 TAPD、Linear、GitHub、CNB 和真實 Agent 組合範例；
7. 保留技術細節，但不要讓每個新讀者一開始就被技術細節淹沒。

---

## 如何貢獻

有價值的貢獻包括：

- 更好的範例；
- 更清楚的文件；
- 更安全的 workflow template；
- 新的專案系統接入；
- 新的 Coding Agent 接入；
- 新的程式碼平台接入；
- dashboard 改進；
- 真實 workflow 測試覆蓋；
- 母語使用者的翻譯審校。

建議先從本地 memory/mock 流程開始，再逐步走向真實系統。
